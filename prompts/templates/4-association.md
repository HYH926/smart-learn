# 关联内化模板 — 步骤4

## 任务

检索知识库中的已有学习记录，将新学概念与已有知识进行对比，建立知识网络。

## 检索

在项目根目录的 `knowledge_store/` 下搜索相关笔记：

1. 用 Glob 列出 `knowledge_store/**/*.md` 文件
2. 用 Grep 在 knowledge_store/ 中搜索关键词匹配
3. 读到匹配的笔记，提取核心概念和关键词
4. 如果 knowledge_store/ 目录不存在，跳过检索（首次使用正常现象）

## 对比维度

对每个相关旧主题，四维对比：

| 维度 | 问题 |
|------|------|
| 问题域 | 解决的是同一个问题吗？还是同一问题的不同阶段？ |
| 可迁移经验 | 已学概念的哪些直觉/经验可以直接帮助理解新概念？ |
| 差异点 | 新概念的哪些特性是已学概念没有的？ |
| 位置关系 | 在知识地图中，两个概念分别在哪？什么关系？ |

## 输出格式

```markdown
## 🔗 知识关联

### {新概念} ↔ {已有概念}

| 维度 | {新概念} | {已有概念} |
|------|---------|-----------|
| 解决的问题 | | |
| 核心方法 | | |
| 适用场景 | | |
| 局限性 | | |

**关联总结**：{一段话总结两者的关系}

---

（对每个相关主题重复）

## 🗺️ 知识网络图

```
{新概念}
  ├── 和 {已有概念1} 的关系：{一句话}
  ├── 和 {已有概念2} 的关系：{一句话}
  └── 下一篇可学：{建议方向}
```
```

## 如果无关联主题

```
## 🔗 知识关联

知识库中暂未发现与「{新概念}」直接关联的主题。
这是你在「{领域}」领域的第一个知识节点。后续学习相关主题时将自动关联。
```
