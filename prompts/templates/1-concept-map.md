# 概念地图模板 — 步骤1

## 任务

将【学习主题】拆解为 5~8 个核心子概念，做到：

1. **子概念数量**：5~8 个。太少说明拆解不够，太多说明粒度太细
2. **按学习依赖排序**：如果理解 B 必须先理解 A，A 必须排在 B 前面
3. **每个概念用一句话说明它解决什么问题**：从实际工程或认知需求出发，不能是"帮助理解X"这种废话
4. **标注难度**：入门 / 中等 / 进阶（不要全标"中等"，要反映真实学习曲线）

## 输出格式（严格遵守）

```
## 📊 概念地图：{主题名}

1. {概念名}（{难度}）→ {一句话解决什么问题}
   ↓
2. {概念名}（{难度}）→ {一句话解决什么问题}
   ↓
3. {概念名}（{难度}）→ {一句话解决什么问题}
   ...
```

箭头 `↓` 表示学习依赖：必须先理解上一个，才能理解下一个。

## 示例

对于一个"消息队列"的主题：

```
## 📊 概念地图：消息队列

1. 同步 vs 异步通信（入门）→ 为什么需要"不等结果"的通信模型
   ↓
2. 消息模型与队列语义（入门）→ 生产者和消费者如何通过队列解耦
   ↓
3. 消息可靠性与确认机制（中等）→ 消息不丢、不重的保障手段
   ↓
4. 消费模型与并发（中等）→ 多消费者如何分配与协调
   ↓
5. 顺序保证与幂等性（进阶）→ 乱序消息的业务影响与应对
   ↓
6. 流处理与事件溯源（进阶）→ 从消息到事件，从单条到流
```
